Create `platform/content-engine/anomaly-detector.js` and its test file.

Required behavior:
- read current and historical KPI rows only from `company_kpi_daily`
- compute 7-day and 30-day baselines excluding the current day
- skip anomaly creation when history support is insufficient
- classify anomalies into `low`, `medium`, `high`, `critical`
- upsert anomaly rows into `company_anomalies`
- set `narration_required` only for `high` and `critical`
- patch MKG silently for `low`/`medium` using KPI/anomaly summaries only
- call Groq narration only after severity is known and only for `high`/`critical`

Implementation constraints:
- no raw connector payloads in prompts, logs, MKG patches, or returned summaries
- keep the detector callable directly so the nightly scheduler in Task 2 can invoke it without redesign
- use deterministic threshold logic, not a custom statistical engine
